1. Introduction

LM‑63 is the long‑standing IES standard describing the plain‑text .IES file format for luminaire photometric data. This 2019 revision clarifies terminology and sets LM‑63 into a “stability” phase while an XML successor (TM‑33) matures. Both remain valid during the transition.

2. Scope

Defines the data fields and ordering for photometric files used for transfer, storage, retrieval and calculation workflows.

3. Key definitions (abridged)

  • Absolute photometry: Candela values measured directly for the test luminaire (lamp lumens not used).
  • Relative photometry: Candela values scaled relative to a reference lamp’s rated lumens.
  • Photometric zero: Direction directly below the source at test position (Type B & C: H=0°, V=0°).
  • Photometric horizontal: “Straight ahead” direction (Type B & C: H=0°, V=90°).
  • Photometric plane: Plane on which data are measured (vertical in Types A & C; horizontal axis for Type B).
  • Horizontal / Vertical angles: Angle grids defining measurement directions per goniophotometer type.
  • Keyword: Square‑bracketed label lines (e.g., [TEST]) that precede the numeric blocks.

4. What changed vs LM‑63‑02 (high level)

  • Clarified definitions (photometric zero/horizontal; absolute vs relative).
  • Replaced <future use> with <file generation type> plus optional [FILEGENINFO].
  • Removed TILT=<filename> option; now use TILT=NONE or TILT=INCLUDE.

5. LM‑63 file layout (essentials)

An LM‑63‑2019 file is plain text with this sequence:

#Line / fieldNotes
1IES:LM-63-2019Mandatory first line.
2…[KEYWORD] valueAny number of defined keywords (see below).
—TILT=NONE or TILT=INCLUDEMarks end of keyword block.
—<#lamps> <lumens/lamp> <mult> <#V> <#H> <type> <units> <width> <length> <height>Core header numbers.
—<ballast factor> <file generation type> <input watts>Header continued.
—<vertical angles>Monotonic 0→… list.
—<horizontal angles>Monotonic 0→… list.
—<candela values>One block per H‑angle; values ordered by V‑angles.

Photometric type 1=Type C, 2=Type B, 3=Type A. Units 1=feet, 2=metres.

5.2 Required keywords (minimum set)

  • [TEST] — test report ID
  • [TESTLAB] — testing laboratory
  • [ISSUEDATE] — file issue date
  • [MANUFAC] — manufacturer

Common useful extras: [LUMCAT], [LUMINAIRE], [LAMPCAT], [LAMP], [OTHER], [FILEGENINFO].

5.3 TILT line

  • TILT=NONE if output does not vary with tilt (skip tilt block).
  • TILT=INCLUDE if tilt multipliers are embedded (see Annex F format).

Absolute photometry: set <lumens per lamp> to -1 and note absolute candela in [OTHER].

5.10 Luminous dimensions

Describe the luminous opening at photometric zero (not the housing size): units (1=ft, 2=m), width, length, height. Orientation is with respect to the zero‑degree photometric plane; length may be less than width depending on orientation conventions.

Angles & candela values

  • <#vertical> and <#horizontal> give list lengths; angle lists are ascending.
  • Candela blocks are provided for each horizontal angle in sequence, each block covering all vertical angles.
  • <multiplier> scales all candela values if not 1.0.

Programming & file conventions (brief)

  • Delimiters may be commas, spaces, multiple spaces, or CRLF.
  • Filenames end with .ies / .IES.
  • Use keywords from the defined list; keep angle lists well‑formed and monotonic.
